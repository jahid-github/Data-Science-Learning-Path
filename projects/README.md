# Projects

This folder is the recruiter-facing execution layer of the repository.

The goal is simple: convert the study roadmap into a small number of strong, reproducible, evidence-backed projects.

## Priority Projects

| Project | Goal | Hiring signal | Status |
|--------|------|---------------|--------|
| [01_data_engineering_job_market_pipeline](./01_data_engineering_job_market_pipeline/) | Build an end-to-end jobs ETL and analytics pipeline | Data engineering, SQL, orchestration, BI | Planned |
| [02_tabular_ml_job_market_intelligence](./02_tabular_ml_job_market_intelligence/) | Train models on job-market data and surface useful insights | Feature engineering, ML evaluation, business framing | Planned |
| [03_computer_vision_defect_detection](./03_computer_vision_defect_detection/) | Detect defects or anomalies in images with measurable performance | CV engineering, deployment thinking, failure analysis | Planned |
| [04_physical_ai_ros2_vision_guided_manipulation](./04_physical_ai_ros2_vision_guided_manipulation/) | Connect perception to robot action in simulation | ROS2, TF2, planning, Physical AI systems thinking | Planned |

## Project Acceptance Standard

Each project should eventually contain:

- `README.md` with setup, architecture, metrics, and results
- source code
- environment or dependency definition
- reproducible commands
- sample output, screenshots, or video links

## Recommended Build Order

1. `01_data_engineering_job_market_pipeline`
2. `04_physical_ai_ros2_vision_guided_manipulation`
3. `02_tabular_ml_job_market_intelligence`
4. `03_computer_vision_defect_detection`

That order keeps the portfolio balanced across broad employability and specialization depth.

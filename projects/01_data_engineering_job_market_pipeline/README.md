# Job Market Data Engineering Pipeline

## Status

`Planned`

## Why This Project Exists

This project is meant to prove end-to-end data engineering ability with a business-relevant dataset: job listings, skills, locations, and salary signals.

It should answer questions such as:

- which tools appear most often in target roles
- which locations have the strongest demand
- which roles overlap with Data, ML, and Physical AI skills

## Target Stack

- Python
- SQL
- PostgreSQL or DuckDB
- dbt or SQL transformation layer
- Airflow, Prefect, or scheduled scripts
- Power BI, Metabase, or Plotly dashboard
- Docker

## Minimum Deliverables

- raw data ingestion from one or more sources
- cleaned and normalized tables
- reproducible transformation pipeline
- documented schema
- dashboard or report with key findings
- setup instructions from a fresh clone

## Strong Portfolio Evidence

- data quality checks
- incremental loads
- orchestration screenshots
- ER diagram
- one short demo video

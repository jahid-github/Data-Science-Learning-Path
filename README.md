# Data, AI/ML, and Physical AI Portfolio

This repository now serves two purposes:

1. a structured learning roadmap and reference library
2. a portfolio workspace for building evidence-backed projects

The target career direction is:

`Data / ML Engineer -> Computer Vision -> Physical AI / Robotics ML Engineer`

The strongest long-term positioning for this repo is not "I collected many resources." It is:

`I can build data systems, train and evaluate ML models, and connect perception to action in real or simulated robotic systems.`

## Portfolio Standard

Every serious project in this repository should eventually include:

- a clear problem statement
- architecture diagram
- reproducible setup steps
- data source description
- evaluation metrics
- failure cases and tradeoffs
- screenshots or short demo video
- honest notes on what is complete vs missing

If a skill is listed in the stack but not backed by code, metrics, or demos, it should be treated as `planned`, not `proven`.

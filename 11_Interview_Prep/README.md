# Interview Prep

Structured preparation for Data Science, ML Engineering, and AI/Robotics roles aligned with this learning path.

---

## 1. Python & Programming

- Data structures: lists, dicts, sets, deques — when to use each
- OOP: classes, inheritance, `__init__`, `@staticmethod`, `@classmethod`
- List comprehensions, generators, decorators, context managers
- Common algorithms: sorting, binary search, BFS/DFS, recursion
- Time and space complexity (Big-O analysis)
- Practice: [LeetCode](https://leetcode.com/) — focus Easy/Medium (arrays, strings, hashmaps)

---

## 2. SQL & Data Engineering

- Aggregations, GROUP BY, HAVING, subqueries, CTEs
- Window functions: `ROW_NUMBER`, `RANK`, `LAG/LEAD`, `PARTITION BY`
- JOINs: INNER, LEFT, RIGHT, FULL, CROSS — when each applies
- Query optimization: indexes, execution plans, partitioning
- Data pipeline design: ETL vs ELT, batch vs streaming
- Schema design: Star vs Snowflake, normalization, denormalization
- Practice: [StrataScratch](https://www.stratascratch.com/), [DataLemur](https://datalemur.com/)

---

## 3. Statistics & Probability

- Descriptive stats: mean, median, variance, standard deviation, skewness
- Probability: Bayes theorem, conditional probability, distributions
- Hypothesis testing: t-test, chi-squared, p-values, confidence intervals
- A/B testing: experiment design, statistical significance, sample size
- Common interview questions: Central Limit Theorem, Law of Large Numbers

---

## 4. Machine Learning

- Bias-variance tradeoff, overfitting, underfitting
- Model evaluation: accuracy, precision, recall, F1, ROC-AUC, RMSE
- Regularization: L1 (Lasso), L2 (Ridge), dropout
- Feature engineering, selection, and importance
- Cross-validation: k-fold, stratified k-fold
- Algorithms to explain clearly:
  - Linear/Logistic Regression
  - Decision Trees, Random Forest, Gradient Boosting (XGBoost)
  - k-Nearest Neighbors, SVM, k-Means, PCA
- When to use which model and why

---

## 5. Deep Learning

- How backpropagation works (chain rule, gradients)
- Vanishing/exploding gradients — how to handle them
- Activation functions: ReLU, sigmoid, softmax — tradeoffs
- Batch normalization, dropout, learning rate schedules
- CNN architecture: convolution, pooling, receptive field
- Transfer learning: when to fine-tune vs feature extraction
- Transformer attention mechanism explained simply

---

## 6. Computer Vision & Robotics (Specialized)

- Object detection: YOLO vs Faster R-CNN — speed vs accuracy tradeoffs
- Image segmentation: semantic vs instance vs panoptic
- Camera calibration, homography, coordinate transforms
- ROS2 concepts: nodes, topics, services, actions, TF tree
- Real-time inference: latency constraints, edge vs cloud tradeoffs
- Sensor fusion: camera + LiDAR + IMU — why and how

---

## 7. System Design for ML/AI

- Designing an end-to-end ML pipeline (data → model → serving)
- Feature stores, model registries, experiment tracking
- Real-time vs batch inference — when to use each
- Designing a recommendation system
- Designing a fraud detection pipeline
- RAG system design: retrieval, chunking, embedding, reranking
- MLOps: CI/CD for models, monitoring, drift detection
- Resources: [Chip Huyen ML System Design](https://huyenchip.com/machine-learning-systems-design/toc.html)

---

## 8. Behavioral & Situational

- STAR method: Situation, Task, Action, Result
- Prepare stories for: "Tell me about a project," "How did you handle failure," "Working with ambiguous data"
- Know your projects deeply — data source, decisions made, results, what you'd do differently
- Questions to ask interviewers about the team's ML stack and data infrastructure
